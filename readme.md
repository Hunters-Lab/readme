# readme

### 说明：

私有文件前加**private**

公共文件前加**public**

### Github仓库命名规范：

1. **库名中不得出现下述规定的字符`-` `\` `@` `!` `#` `$` `%` `^` `&` `*` `()` `[]` `{}` `|` `\` `;` `:` `''` `’` `，` `。` `《` `》` `<` `>` `·` `~` 。**
2. 库名应尽量避免使用 `名.名`的形式。
3. 库名应尽量使用**英文**，**禁止使用中文字符**。一般情况下，库名中出现的各个单词的首字母应使用大写。各个单词不能使用连接符 `-`连接；如有必要，**应使用下划线 `_` **。
4. 缩写的单词一律使用大写。

### 举例：

#### private_Energy-Conservation-and-Emission-Reduction-Competition_xxx    

#### public_Power-section

**注：XXX为相关负责人**
